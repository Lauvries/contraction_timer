Host the web UI (HTML/JS/CSS) somewhere. Supabase only hosts your database/APIs — not this static app.

Quickest free option — GitHub Pages (workflow already in repo):

  1) Create a new empty repository on GitHub (no README).
  2) In this folder run:
       git remote add origin https://github.com/YOU/REPO.git
       git push -u origin main
  3) On GitHub: Settings → Pages → Build and deployment → Source: GitHub Actions.
  4) Open the Actions tab; when the deploy workflow finishes, Pages shows the live URL
     (typically https://YOU.github.io/REPO/).

  Then in Supabase → Authentication → URL Configuration, set Site URL to that live URL
  and add the same URL under Redirect URLs (wildcard ok: https://YOU.github.io/REPO/**).

Supabase project: enable Auth → Providers → Anonymous (this app uses anonymous sign-in).
Schema is already applied if you used the Supabase MCP migration.

Netlify (free): create empty site; optional workflow .github/workflows/deploy-netlify.yml
needs secrets NETLIFY_AUTH_TOKEN, NETLIFY_SITE_ID, SUPABASE_URL, SUPABASE_ANON_KEY; or
link the repo in Netlify UI with build env vars (see netlify.toml).

Local: npm run build injects supabase-config.js when SUPABASE_URL / SUPABASE_ANON_KEY are set.
Local test: python3 serve.py (http://, not file://).
