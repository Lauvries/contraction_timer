Host the web UI (HTML/JS/CSS) somewhere. Supabase only hosts your database/APIs — not this static app.

GitHub Pages (workflow: .github/workflows/deploy-pages.yml)

  1) On https://github.com/new create a repository (e.g. MTGapp), empty — no README, no .gitignore.
  2) In this project folder:
       git remote add origin https://github.com/YOUR_USER/MTGapp.git
       git push -u origin main
     (Use SSH git@github.com:... instead if that is how you authenticate.)
  3) GitHub → your repo → Settings → Pages → Build and deployment → Source: GitHub Actions.
     (Pick “GitHub Actions”, not “Deploy from a branch”.)
  4) Actions tab → open “Deploy static site to Pages” → wait for green; refresh Settings → Pages
     for the public URL (usually https://YOUR_USER.github.io/MTGapp/).

  Supabase → Authentication → URL Configuration: set Site URL to that Pages URL; add Redirect URL
  https://YOUR_USER.github.io/MTGapp/** (and http://localhost:PORT/** for local serve.py if you use it).

Supabase project: enable Auth → Providers → Anonymous (this app uses anonymous sign-in).
Schema is already applied if you used the Supabase MCP migration.

---

Netlify (optional): workflow .github/workflows/deploy-netlify.yml runs only when you start it
manually (Actions → Deploy to Netlify → Run workflow). Add the secrets above first, or use
Netlify’s own “connect GitHub repo” flow instead.

Local: npm run build injects supabase-config.js when SUPABASE_URL / SUPABASE_ANON_KEY are set.
Local test: python3 serve.py (http://, not file://).
